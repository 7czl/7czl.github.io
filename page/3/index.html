费曼学习法

Posted on 2023-03-28 Edited on 2023-03-29

明确想要学习的内容，每一个学习目标都必须明确且细化，不然就会迷失在不同话题的跳转中，而无法深入理解。
向一个六年级的学生解释这个话题，这一步会将复杂的话题分解为几个基础部分，最终目标是对这个话题有更进一步的理解。你的解释需要简短直白。这个解释的过程可以有几种实现形式。 找个真正的聆听者，或是对着镜子演讲。找到适合自己的方式即可。
复盘解释的过程，找到没有覆盖到的地方，或是那些没有用简单的语言阐述的部分。
提高或者是改进解释。修改那些需要被改进的地方。譬如使用类比，对术语的进一步说明，确保解释是基于事实的并且是有基础材料的支撑。如果不能够使解释足够简单，回归基础材料来强化对概念的理解，直到可以使用足够简单的语言来描述此话题。
重复上述四步直到你已经完全的掌握了这个话题。

调用约定

Posted on 2023-03-28 Edited on 2023-03-29 In asm

调用规则涉及函数如何传参，如何传递返回值，函数如何管理栈帧，简而言之，调用规则影响C/C++代码编译到汇编语言。没有调用规则的限制，不同的编译器编译的程序不可能进行交互。
主要的三种STDCALL CDECL和FASTCALL，对C++来说还有一种叫thiscall的调用规则。
X86_64有一套单独的调用规则。
传值
func(a, b) ->
push a, push b, call func
or
push b, push a, call func

返回值
被调用函数需要在ret指令前处理返回值。存储到指定的地方
清理栈帧
使用栈传递参数的话需要由调用者，或被调用者来完成清理工作。恢复栈寄存器rbp/rsp
调用函数（调用者
父函数，子过程的调用者

wsl2安装fedora37并开启Gnome图形界面

wsl2可以指定安装特定的发行版，譬如fedora
确保wsl2是最新的，去fedora container base下载x86_64的镜像
https://kojipkgs.fedoraproject.org/packages/Fedora-Container-Base/37/1.7/images/
下载tar.xz后解压缩layer.tar到任意目录重命名为fedora37.tar
在windows下创建一个目录用于fedora

但会遇到问题，wsl开始支持systemd启动见如下链接
https://devblogs.microsoft.com/commandline/systemd-support-is-now-available-in-wsl/

编辑/etc/wsl.conf 添加

重启wsl

运行 wsl –shutdown

xrdp启动后需要添加防火墙规则和编辑selinux例外。（我这里的情况都是默认关闭的）

Considering Rust

Posted on 2023-03-28 Edited on 2023-03-29 In rust language

https://www.youtube.com/watch?v=DnT-LUQgc7s

这是关于Jon Gjengset的视频的笔记

  • 为什么使用rust

  • rust的卖点

  • 和Java/Cpp/Python的比较

  • rust的缺点:
    缺少运行时意味着缺少运行时的反射，虽然可以使用GDB/LLDB来监测运行时的行为，但是不方便。在运行时debugging时缺少探测辅助。
    编译时间较长。Rust缺少预编译的二进制，譬如.so或.dll这种动态链接库。因为泛型函数的存在，当你的函数需要调用库的函数，而库函数是泛型的时候，需要从源码编译这个泛型函数所在的库。
    当需要使用外部非开源库时，写胶水代码是一个苦力活。
    有些开源的三方库只支持Linux/OSX，对Win支持的寥寥，譬如异步库，需要和操作系统打交道的特定情况，这种支持比较缺乏。

  • 长期生命力
    这种考虑源于我们在学习，采用一门新语言到项目中时对这门语言的生命力的考量。
    Rust是未来的一个很好的选择，原因如下

    1. 程序员最喜爱的语言，Stackoverflow调查，持续四年。
    2. 大公司采用的较多。Mozilla，Fb，Amazon， Goole，MS
    3. 语言的可交互性好，可以渐进式的重写部分已存在项目的模块。
    4. 语言峰会已持续10年以上，覆盖更多大城市，更多的开发者和对语言感兴趣的人的加入，社区的壮大。Rust的生态关注程序员体验，JG说他很愿意使用Rust来重写旧项目，这种感觉他在别的语言从未体验过。Rust编译器提供了更友好的提示信息，当程序员出现错误时，编译器像一个教练一样教你如何写出/修改正确的代码。当大规模重构的时候Rust编译器更能够体现其优越的错误提示。
