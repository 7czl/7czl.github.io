GCC和LLVM的区别

Lex parser 生成AST语法树 转化为 GCC的 RTL(Register Transfer Language)，然后由后端生成机器原生代码。
标准的工作流 C source -> GCC C-frontend -> RTL -> GCC X86 backend -> X86 machine code
预处理 -> 汇编->.S -> 汇编器-> -> .o -> 链接器 -> .elf or .so
LLVM是GCC的一个替代，可以生成LLVM-IR，这种格式类似于RTL，编译器前端支持不同的语言源代码编译到LLVM-IR由LLVM-Platform backend 来生成目标代码

博客建立和自动化部署

Posted on 2023-03-30 Edited on 2023-04-08

记录下建站的过程:

  1. hexo 生成静态站点 hexo init
  2. 博客的根目录跟踪的是博客仓库地址。
  3. _post跟踪的是文章仓库地址_，这样不同机器只需要拉取最新文章即可在本地使用hexo g 生成静态页面。
  4. 配置博客主题，设置自动添加分类和一件部署ghpages config.yml 。
  5. obsidian中设置模板，方便自动添加元数据，已设置快捷键ctrl+k。

[!NOTE1]
npm install –save 和–save-dev的区别是这个包安装是否仅和开发有关，还是它是程序运行所必须的依赖项。

终端和桌面管理器还有shell的区别
WM和DE是高度关联的

对linux来说
基于gtk3:
gnome-shell use Mutter
cinnamon use Muffin
mate use marco
Unity use Compiz
XFCE4 use XFWM4
KDE Plasma5 use kwin_x11 or kwin_wayland
桌面环境的介绍
桌面环境的自定义包括窗口管理器，或混成器，任务栏和应用程序，通常包括终端模拟器，文件管理器和文本编辑器

包含桌面管理与显示服务器与混成器的有:Mutter，Muffin，Compiz
单独的桌面管理器bspwm，i3，Xnomad，dwm
不同桌面管理器支持的布局方式： stacking，Tiling，floating

桌面环境的定制包含哪些内容？
主题，颜色方案，墙纸，文件管理器中的上下文菜单，窗体动画，光标和图标集合。
